Featured Project

Smart Store Monitoring System

Real-time CCTV integration, POS correlation, and intelligent analytics for retail operations

Overview

The Smart Store Monitoring System is an integrated solution that combines CCTV surveillance, point-of-sale data, and intelligent analytics to provide store owners with comprehensive visibility into their operations. This system transforms raw video feeds and transaction data into actionable insights, helping businesses reduce shrinkage, optimize staffing, improve customer experience, and enhance security.

Built with a cross-platform architecture using Flutter for the frontend and a robust backend service layer, this system operates seamlessly across desktop, mobile, and cloud environments with full offline-first capabilities.

Key Features

📹

Multi-Camera CCTV Integration

Connect multiple RTSP/HTTP camera streams with synchronized playback, DVR-style controls, and grid/single views. Supports local video files and live streaming with automatic reconnection.

🛒

POS Transaction Correlation

Automatically link video footage to point-of-sale transactions by timestamp. Review any transaction with synchronized video playback for dispute resolution and fraud detection.

🚨

Real-Time Alerts & Notifications

Instant notifications for suspicious activity, motion detection, door sensor triggers, checkout anomalies, and inventory discrepancies. Multi-channel alerting via email, SMS, and webhooks.

📊

Analytics Dashboard

Comprehensive metrics including footfall counting, customer dwell time, conversion rates, peak hours analysis, heatmaps, and shrinkage tracking with visual charts and trend analysis.

🔍

Forensic Playback & Search

Time-based video search with frame-accurate scrubbing, bookmarking critical events, exporting clips, and generating incident reports with embedded screenshots and metadata.

👥

Role-Based Access Control

Secure multi-user system with granular permissions for administrators, store managers, cashiers, and auditors. Each role sees only relevant data and controls.

☁️

Offline-First with Cloud Sync

Local SQLite persistence ensures operation during network outages. Automatic cloud synchronization when connectivity is restored with conflict resolution and versioning.

🌐

Cross-Platform Compatibility

Runs on Windows, macOS, Android, and iOS with responsive UI adaptation. Desktop apps for management, mobile apps for remote monitoring and alerts.

How It Works

01

Data Ingestion

The system continuously ingests multiple data streams: RTSP/HTTP camera feeds, POS transaction events via webhooks or database polling, and sensor data from door contacts, motion detectors, and weight scales.

02

Timestamp Correlation

All incoming events are timestamped with synchronized clocks. The correlation engine links POS transactions to corresponding video frames, sensor triggers to camera footage, and creates a unified timeline of all store activity.

03

Analysis & Detection

The analytics engine processes video streams and transaction data to detect patterns: people counting via motion detection, dwell time calculation, anomaly detection for unusual checkout behavior, inventory variance alerts, and suspicious activity flags.

04

Alerting & Dashboard

Critical events trigger real-time alerts sent to configured channels. All metrics flow into the live dashboard showing current camera feeds, recent transactions, active alerts, and key performance indicators updated in real-time.

05

Storage & Sync

Video segments and events are stored locally in a versioned SQLite database for instant access and offline reliability. Background sync processes upload data to cloud storage for long-term retention, cross-store analytics, and remote access.
